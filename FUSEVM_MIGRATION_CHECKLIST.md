# stryke → fusevm Migration Checklist

Goal: retire stryke's own `vm.rs` (~11.8k lines) + `jit.rs` (~5.5k lines); all
execution runs on `fusevm` via `fusevm_native.rs` (bytecode→fusevm lowering +
`native_ext_handler` host delegation).

## Current state (last updated end of Phase 36)

- **Ops lowered:** 115 / 351 `Op` variants have native lowering arms.
- **Corpus (`bench/*.pl` + `examples/*`): 12/12 fully native, all parity-clean.**
- **Lib suite:** 4200 tests passing.
- **Active measurement corpus:** `parity/cases/*.pl` (1,506 files), generated by
  `parity/gen_parity_2k.py` (shared op template → each phase clears the next wall
  for ~all files at once).

### How to measure
```sh
# rank the first uncovered op across a sample (every 6th file — ~250 samples)
for f in $(find parity/cases -name "*.pl" | sort | awk 'NR%6==1'); do
  env STRYKE_FUSEVM_ONLY=1 STRYKE_FUSEVM_TRACE=1 ./target/debug/stryke "$f" 2>&1 >/dev/null \
    | grep -m1 FUSEVM_UNCOVERED | awk '{print $2}'
  echo "${op:-__NATIVE__}"
done | sort | uniq -c | sort -rn
# disasm a file's full op set (stderr!): ./target/debug/stryke --disasm FILE 2>&1 >/dev/null
# env flags: STRYKE_FUSEVM_ONLY=1 (force native, fall back if uncovered),
#            STRYKE_FUSEVM_STRICT=1 (uncovered = hard error), STRYKE_FUSEVM_TRACE=1
```

### Per-op workflow (one phase)
1. `grep -n "Op::Name" src/vm.rs` → read the exact semantics (port faithfully).
2. Verify any helper methods are reachable (scope/VMHelper/value pub(crate)/pub).
3. Add `nops::NAME` const + handler in `native_ext_handler` + lowering arm in
   `try_run_native`. Multi-operand ops push operands via `LoadInt` and pop them.
4. Add a `#[test]` using `assert_parity_int/str` (must run natively — confirm with
   a `--disasm`/TRACE probe that the source actually emits the op).
5. `cargo test --lib fusevm_native::`, then `cargo build --bin stryke`, re-rank,
   then **full `cargo test --lib`**, commit, re-sample parity.

## NEXT WALL — `Op::Call` (parity corpus #1, 237/244)

Dynamic name-based dispatch (builtins + user subs). **Hardest remaining op.**
- Routes through `vm.rs::vm_dispatch_user_call` (3061) — deeply VM-coupled: AOP
  intercepts, profiler, debugger, fib-like fast path, stack_args, return types.
- No clean `BuiltinId::from_name`; builtin dispatch goes through
  `builtins.rs::try_builtin` (1173) which is entangled with Expr-based arg eval.
- Native path has pre-evaluated args on the stack → need a path that takes
  (name, Vec<StrykeValue>, wantarray). `call_named_sub` covers user subs; need the
  builtin-by-name half. Likely extract a shared `VMHelper` method (single-source
  with the AST `FuncCall` path at vm_helper.rs:11095) — design carefully, do NOT
  rush (silent-divergence risk).

## Watch-outs (learned this session)

- **Registry-handle flatten:** non-scalar StrykeValues ride the fusevm stack as
  `Value::NativeFn(reg id)`. Anywhere vm.rs does `val.to_list()`, the native
  handler must use `pop_stryke(vm).to_list()` (handles fusevm-Array, registry
  handle, and scalar uniformly). A manual `match Value::Array` nests the handle.
- **Overloaded operators:** BitAnd/Or/Xor have set + sketch overloads before the
  integer path; StrCmp has operator-overload. Mirror vm.rs, don't map to fusevm's
  plain integer/native ops.
- **Sub bodies are dead on native** (CALL_SUB delegates to `call_named_sub`); the
  lowering stops at the earliest `chunk.sub_entries` ip, so sub-body-only ops never
  need arms.
- **Slots are `u8`; the Extended `arg` field is `u8`** — pass larger counts/operands
  via `LoadInt` then pop, never via `arg`.
- Verify an op actually fires natively before trusting a parity test (a fallback
  makes default==fusevm trivially, hiding an untested handler).
