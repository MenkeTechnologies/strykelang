## Suggested fix order

1. **Bulk doc cleanup pass** — most `doc` failures fix together: scan `src/lsp.rs`, define helper subs inline (`sub do_work { … }`) and concrete sample data (`my @data = (…)`) at the top of each multi-statement snippet that references them.
2. **`pair*` / `uniq` + `%hash`** — the lexer treats `%` after a known list-builtin name as modulus instead of hash-sigil. Needs lexer context-awareness; see also `parse_named_expr` arm. Cascades to `Illegal modulus zero` and to many doc rewrites.
3. **`to_toml` / `to_yaml` nested HASH** — the codec impls fall back to `json_encode` for nested values which rejects HASH. Add native hash → TOML/YAML map serialization.
4. **Socket builtins** — register `sockaddr_in`/`inet_aton`/`inet_ntoa` as native built-ins (no `use Socket`).
5. **`fn ($arr, $hash)` Perl-style sigs** — extend `parse_sub_sig_or_prototype_opt` to accept array/hash params for `fn` (matches Perl 5.20+ signatures).
6. **`open '+<'` read-write mode** — small builtin fix.
7. **`pack`/`unpack` `B` format** — bit-string format, currently unsupported.
8. **Internal panic** — trace the panic backtrace; dedicated bug.
